# Partnership Source Opportunities

Some valuable roaming and hub datasets are not public enough for direct
connectors. They require explicit access and redistribution terms.

| Source owner   | Expected data                                         | Likely access route               | Minimal export fields                                                    | Redistribution questions                                               | Expected benefit                     |
| -------------- | ----------------------------------------------------- | --------------------------------- | ------------------------------------------------------------------------ | ---------------------------------------------------------------------- | ------------------------------------ |
| Hubject        | Network participants, roles, hub IDs, OCPI/eMI3 hints | Partnership or written permission | technical ID, role, country, party ID, organization, status, last update | Can derived observations be redistributed? Can conflicts be published? | Broad European roaming coverage      |
| Gireve         | Participants, roles, roaming IDs                      | Partnership or written permission | technical ID, role, country, party ID, organization, status, last update | Same as above                                                          | French and European roaming coverage |
| e-clearing.net | Network registrations                                 | Partnership or written permission | network party ID, role, organization, status                             | Same as above                                                          | German and regional roaming coverage |
| ENAPI          | National roaming/platform membership                  | Partnership or written permission | party ID, role, organization, status                                     | Same as above                                                          | National network coverage            |

Proposed collaboration message:

```text
We maintain an open eMobility identifier registry that distinguishes official
assignments from secondary observations. We would like to publish derived
network-membership observations with source attribution, confidence, and conflict
diagnostics. Could you provide a machine-readable export and clarify
redistribution terms?
```
